ABI nativa

Interop C via OM

Bibliotecas C entram como ABI nativa no LLVM. O usuario escreve Snask; o OM tenta deduzir contratos de recursos a partir dos headers.

Scanner

snask om scan SDL2/SDL.h --lib sdl2 --output /tmp/sdl2.generated.om.snif

Import conceitual

import_c_om "SDL2/SDL.h" as sdl2

Modelo

  1. O scanner le o header com Clang.
  2. Heuristicas classificam construtores, destruidores, metodos e funcoes seguras.
  3. O contrato final entra no compilador.
  4. LLVM emite chamada nativa; recursos opacos podem ser registrados no runtime OM.

Limites atuais

Callbacks, structs complexas por valor, ownership ambiguo e varargs avancado ainda exigem novas regras ou patches `.om.snif`.
