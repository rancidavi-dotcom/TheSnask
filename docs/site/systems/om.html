OM-Snask-System

O OM-Snask-System e o sistema unico de memoria, recursos e interop nativa da Snask.

Ideia central

Em vez de espalhar `malloc`, `free`, `destroy`, ownership manual e bindings gigantes pelo codigo, a Snask tenta deduzir o ciclo de vida a partir da intencao.

Frase guia: em vez de voce escrever o contrato, o Snask deve deduzir o contrato.

O que pertence ao OM

Zonas

Delimitam vida, agrupam alocacoes e recursos.

Arenas

Direcao para alocacao temporaria rapida e limpeza em massa.

Recursos externos

Handles C ou runtime com destrutores conhecidos.

Contratos C

Gerados automaticamente por scanner e corrigidos por `.om.snif` quando necessario.

Zona simples

class main {
    fun start() {
        zone "frame" {
            let message = "dentro da zona"
            print("{message}\n")
        }
    }
}

Status

`@unsafe` para chamadas restritas ja e estavel. O formalismo completo do OM ainda e experimental: escape analysis, borrow checking e zone depth formal continuam no roadmap.
