Capítulo 6

Perfil Systems e Baixo Nível

Quando o desempenho e o controle de hardware superam a necessidade de conveniência, entra em cena o Perfil Systems do Snask.

Ativando o Perfil Systems

Por padrão, o Snask compila no perfil humane. Para ativar o modo de sistemas, você deve informar ao compilador via flag: 

snask build app.snask --profile systems --output app

No perfil systems, o runtime pesado é removido, o binário fica menor e você ganha acesso a primitivas de hardware.

O Bloco @unsafe

Seguindo a filosofia de linguagens seguras modernas, operações perigosas (como aritmética de ponteiros) são isoladas em blocos @unsafe. Isso serve como um "contrato de consciência" para o desenvolvedor. 

@unsafe {
    let p: ptr = mem_alloc(1024)
    mem_write_u8(p, 0, 0xFF)
    mem_free(p)
}

Aritmética de Bits e Casts

Em sistemas, cada bit conta. O Snask oferece funções de alta performance para manipular bits sem a verbosidade de C. 

let flags = 0b10101010
if bit_test(flags, 7) {
    print("Bit de status ativo!\n")
}

let byte: u8 = as_u8(512) // Truncamento explícito para 8 bits

Por que usar Systems?

  • Emuladores: Para modelar barramentos de memória e CPUs reais.
  • Drivers: Para conversar com hardware via mapeamento de memória.
  • Sistemas Operacionais: O Snask foi desenhado para permitir a escrita de microkernels.
  • Binários Minimalistas: Quando você precisa de um executável de poucos KBs.
