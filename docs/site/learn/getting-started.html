Capítulo 1

Introdução ao Snask

Snask é uma linguagem de programação moderna, compilada via LLVM, desenhada para ser "Humana por padrão, Systems quando necessário".

A Filosofia Snask

Diferente de linguagens que te forçam a escolher entre "produtividade com Garbage Collector" (como Python/JS) ou "controle manual perigoso" (como C/C++), o Snask introduz o conceito de Perfis Adaptativos.

  • Perfil Humane: Focado na experiência do desenvolvedor. Erros amigáveis, automação de memória via zonas e segurança total.
  • Perfil Systems: Ativa primitivas de baixo nível, acesso a ponteiros e operações de bits, ideal para emuladores e kernels.

Instalação Profissional

O Snask depende do backend LLVM 18 para gerar código nativo de alta performance. O script de instalação oficial cuida de configurar o toolchain para você.

# Instalação automatizada para Linux
curl -fsSL https://raw.githubusercontent.com/rancidavi-dotcom/TheSnask/main/install.sh | bash

# Configure o PATH (adicione ao seu .bashrc ou .zshrc)
export PATH="$HOME/.snask/bin:$PATH"

# Verifique a saúde da instalação
snask doctor

Anatomia de um Programa Snask

Um programa executável sempre começa em uma class main com uma função start(). Diferente de C, não usamos int argc, char** argv na assinatura da main; os argumentos são acessados via builtin args().

class main {
    fun start() {
        print("Olá Mundo!\n")
    }
}

Por que class main? Para garantir que o compilador possa orquestrar o ciclo de vida inicial do programa e as zonas de memória raiz de forma determinística.

O Ciclo de Build

O Snask compila seu código diretamente para binários nativos (AOT - Ahead of Time). Ele não usa uma máquina virtual (VM) como Java ou um interpretador como Python.

# Compilação padrão (Humane)
snask build hello.snask --output hello

# Compilação para sistemas (sem runtime pesado)
snask build app.snask --profile systems --output app
