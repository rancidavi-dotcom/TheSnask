SPS

Projetos Snask

O SPS, Snask Project System, usa `snask.snif` para definir metadados, entrada, build, dependencias e scripts.

Criar projeto

mkdir my_app
cd my_app
snask init
snask build
./my_app

Manifesto minimo

{
  package: { name: "my_app", version: "0.1.0", entry: "main.snask" },
  dependencies: {},
  build: { opt_level: 2, profile: "humane" },
}

Perfis

PerfilUso
humaneApps, CLI, aprendizado e runtime completo.
systemsEmuladores, binarios, memoria crua controlada e baixo nivel.

Lockfile

`snask.lock` registra versoes e hashes de dependencias. O lockfile ja tem leitura/escrita e validacao basica testada; registry e resolucao avancada ainda amadurecem.
