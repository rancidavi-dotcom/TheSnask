Capítulo 2

Variáveis e Sistema de Tipos

Snask é uma linguagem de tipagem estática com inferência poderosa. No Snask, a clareza sobre ownership e mutabilidade começa na declaração da variável.

Imutabilidade por Padrão

Seguindo as melhores práticas de engenharia de software modernas, no Snask tudo é imutável por padrão. Isso evita bugs de "efeito colateral" onde uma variável muda de valor inesperadamente. 

class main {
    fun start() {
        let pi = 3.1415
        // pi = 3.14  <-- Erro de compilação! let é imutável.
    }
}

Mutabilidade Explícita

Se você precisa que um valor mude ao longo do tempo, deve usar a palavra-chave mut. Isso deixa claro para quem lê seu código que aquele estado é volátil. 

mut contador = 0
while contador < 10 {
    contador = contador + 1
}

Constantes de Módulo

Diferente de let, que é uma "ligação imutável" dentro de um escopo, const define valores que são conhecidos em tempo de compilação. 

const VERSAO_API = "1.0.4"
const MAX_CONEXOES = 100

Sistema de Tipos

O Snask possui tipos primitivos robustos. Embora o compilador consiga inferir o tipo na maioria das vezes, você pode ser explícito para aumentar a segurança.

TipoDescriçãoPerfil Recomendado
strString UTF-8 dinâmica.Humane
floatNúmero de ponto flutuante 64-bit.Humane / Systems
boolValor booleano (true/false).Ambos
u8 / i8Inteiros de 8-bit (Byte).Systems
ptrPonteiro de memória crua.Systems (@unsafe)

Anotação de Tipo: Use nome: tipo para declarar tipos manualmente. 

let nome: str = "Snask"
let idade: float = 25.0
let raw_data: u8 = as_u8(255)

Interpolação de Strings

O Snask facilita a exibição de dados com uma sintaxe de interpolação limpa dentro de aspas duplas. 

let user = "Davi"
let score = 95
print("Usuário {user} atingiu a pontuação {score}!\n")
