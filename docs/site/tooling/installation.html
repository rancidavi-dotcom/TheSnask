Instalação

Escolha o método de instalação compatível com seu sistema. O Snask oferece pacotes nativos para Arch Linux (AUR) e Debian/Ubuntu (.deb), além do instalador universal e compilação manual.

Arch Linux (AUR)

Pacote oficial no AUR. Compila o Snask e o LSP a partir do código fonte.

yay -S snask
# ou usando paru:
# paru -S snask

Após instalar, o binário principal estará em /usr/bin/snask e o LSP em /usr/bin/snask-lsp.

AUR compilação: a primeira instalação baixa as dependências e compila o LLVM 18 + Snask. Pode levar alguns minutos. Certifique-se de ter base-devel, git e rust instalados.

Ubuntu / Debian (.deb)

Pacote pré-compilado disponível no GitHub Releases. Contém o binário principal e o LSP.

# Baixar a última versão
curl -fsSL https://github.com/rancidavi-dotcom/TheSnask/releases/latest/download/snask-amd64.deb -o snask.deb

# Instalar
sudo dpkg -i snask.deb

# Verificar instalação
snask doctor

O pacote .deb registra o binário em /usr/bin/snask e o LSP em /usr/bin/snask-lsp. Dependências (LLVM 18, GTK3, zlib, sqlite) são listadas como dependências do pacote.

Instalador Universal (Linux)

Script portátil que detecta a distribuição, instala dependências, compila e configura o Snask no diretório home.

curl -fsSL https://raw.githubusercontent.com/rancidavi-dotcom/TheSnask/main/install.sh | bash

O instalador coloca os binários em ~/.snask/bin/ e pergunta se deseja adicioná-los ao PATH e configurar o Neovim.

Compilação Manual

Para desenvolvedores que preferem compilar do código fonte diretamente:

Pré-requisitos

# Arch Linux
sudo pacman -Syu --needed base-devel git rust cargo llvm18 llvm18-libs clang18 lld18 pkgconf gtk3 zlib sqlite

# Ubuntu/Debian
sudo apt update
sudo apt install build-essential git curl ca-certificates pkg-config rustc cargo \
  clang-18 llvm-18 llvm-18-dev lld-18 libclang-18-dev \
  libgtk-3-dev zlib1g-dev libsqlite3-dev

Compilar e instalar

git clone https://github.com/rancidavi-dotcom/TheSnask.git
cd TheSnask
cargo build --release

# Instalar manualmente
mkdir -p ~/.local/bin
cp target/release/snask ~/.local/bin/
cp target/release/snask-lsp ~/.local/bin/

# Verificar
snask doctor

Pós-instalação

Após instalar, execute o comando de verificação e configure o ambiente:

# Verificar se tudo está funcionando
snask doctor

# Configurar Neovim (instala plugin snask.nvim)
snask setup

# Iniciar um novo projeto
snask init meu_projeto
cd meu_projeto
snask build
LSP: o servidor LSP (snask-lsp) é instalado automaticamente junto com o Snask. Use snask setup para configurar o plugin Neovim automaticamente.

LLVM 18

O Snask usa o backend LLVM 18 via inkwell / llvm-sys. Se sua distribuição instala o LLVM em caminhos não padrão, configure as variáveis de ambiente:

export LLVM_CONFIG_PATH=/usr/bin/llvm-config-18
export LLVM_SYS_180_PREFIX=/usr/lib/llvm-18
export SNASK_CLANG=/usr/bin/clang-18
export SNASK_LLC=/usr/bin/llc-18

# Compilar novamente
cargo build --release
Arch Linux: no Arch, o LLVM 18 é compilado apenas com bibliotecas compartilhadas (.so). O Snask detecta isso automaticamente e usa a feature force-dynamic do llvm-sys.
