OM Memory System

O sistema de gerenciamento automático de memória baseado em zonas. Sem GC, sem leaks, sem pauses.

O OM Memory System permite que você nunca se preocupe com malloc/free no perfil Humane, e ainda oferece controle total via @unsafe quando necessário no perfil Systems.

Perfis: Humane vs Systems

A mesma linguagem, dois níveis de abstração. Escolha o perfil certo para cada parte do seu projeto.

$ snask explain S1002
╔══════════════════════════════════════════╗
║  S1002 — Tipo incompatível em operação  ║
╚══════════════════════════════════════════╝

Contexto:
  A operação `+` espera ambos os operandos
  do mesmo tipo, mas recebeu:
    - Esquerda: u32
    - Direita:  String

Ocorreu em:
  exemplo.snask:7:14

Possíveis causas:
  1. String não convertida para número
  2. Variável declarada com tipo diferente
  3. Uso acidental de `+` em vez de `++`

Sugestão:
  Use `parse_int()` para converter a String
  antes da operação aritmética.

  let x: u32 = parse_int(minha_string) ou 0

Documentação:
  https://snask.dev/reference/diagnostics#S1002
