NOVO

Iniciando a Era Self-Hosted do Snask

Começamos a maior atualização do Snask: tornar a stdlib totalmente self-hosted. O print é o primeiro passo — criamos o módulo stdio em Snask puro (src/stdlib/stdio.snask), removendo o código C do runtime.

Próximos passos: Módulos stdlib modulares (um .snask por módulo), auto-import da stdlib, e substituição gradual de todas as funções C por código Snask compilado. Apenas 4 intrínsecos irreducíveis restarão no core!

# Features

🛡️

Perfis Adaptativos

Humane + Systems na mesma linguagem. Comece com gerenciamento automático de memória e ative controle de hardware apenas onde precisar. Sem GC, sem runtime pesado.

🧠

OM Memory System

Gerenciamento automático de memória baseado em zonas. Recursos são registrados em regiões e liberados ao final da zona. Sem collector, sem pauses, sem leaks.

🔌

C Interop nativa

Compilação via LLVM permite chamar bibliotecas C diretamente. Sem transpiração, sem binding generation. Para casos seguros, a interop é direta e eficiente.

📄

SNIF Format

Formato de dados nativo da Snask. Serialização binária tipada para comunicação entre processos e persistência de estruturas complexas.

# Quick Start

1. Instale

curl -fsSL https://raw.githubusercontent.com/rancidavi-dotcom/TheSnask/main/install.sh | bash
export PATH="$HOME/.snask/bin:$PATH"
snask doctor

2. Crie um projeto

mkdir meu-app && cd meu-app
snask new app
# Cria: main.snask, snask.toml, .gitignore

3. Compile e execute

snask build main.snask
./main
Precisa de ajuda? Leia o guia de introdução ou acesse os exemplos prontos.
