Type system

Sistema de Tipos Snask

O sistema de tipos da Snask combina um núcleo estável de primitivos e coleções genéricas com tipos nominais e um subsistema de tipos de baixo nível para o perfil systems. Esta página documenta cada categoria, seu uso e status atual.

Tipos Primitivos

Os quatro tipos primitivos formam a base de toda a linguagem. Eles estão disponíveis em todos os perfis de compilação.

Tipo Descrição Literais Exemplo
int Inteiro com sinal do tamanho da palavra da máquina 42, -7, 0xFF, 0b1010 let x: int = 10
float Ponto flutuante de 64 bits (IEEE 754) 3.14, 1.5e-10 let pi: float = 3.14159
str String UTF-8 imutável "ola", "{x}\n" let s: str = "Snask"
bool Valor booleano true, false let ok: bool = true

Detalhes: int

O tipo int é a representação nativa de inteiro com sinal. Em plataformas de 64 bits, é equivalente a i64; em plataformas de 32 bits, a i32. Operações aritméticas entre dois int produzem int. Mistura com float segue as regras de coerção numérica.

Detalhes: float

O tipo float segue o padrão IEEE 754 de precisão dupla (64 bits). Literais devem conter um ponto decimal ou notação científica.

Detalhes: str

Strings são sequências de bytes codificadas em UTF-8. São imutáveis por padrão. O runtime garante que strings sejam terminadas em nulo para compatibilidade com C. A interpolação "{expr}" converte automaticamente qualquer expressão para string.

Detalhes: bool

O tipo bool tem dois valores: true e false. Os operadores lógicos and, or e not operam exclusivamente sobre booleanos. Comparações como ==, !=, <, etc. produzem bool.

Tipos Systems

O perfil systems expõe tipos de tamanho fixo para programação de baixo nível. Eles são usados pelas primitivas NES, operações de bits e interface com hardware. O layout e ABI ainda estão em fase de fechamento.

Coleções Genéricas

Snask tem duas coleções genéricas nativas. Ambas suportam literais com tipo inferido, indexação e atribuição.

list<T>

Listas são sequências ordenadas e homogêneas de elementos do tipo T. O acesso é feito por índice numérico começando em 0. 

let vazia: list<int> = []
let nums: list<int> = [1, 2, 3, 4, 5]
let primeiro = nums[0]    // 1
nums[2] = 42              // [1, 2, 42, 4, 5]

let nomes: list<str> = ["Ana", "Bia", "Carlos"]
for nome in nomes {
    print("{nome}\n")
}

dict<K, V>

Dicionários são mapas de chave-valor com chaves do tipo K e valores do tipo V. As chaves devem ser comparáveis com ==. 

let vazio: dict<str, int> = {}
let idades: dict<str, int> = { "Ana": 25, "Bia": 30 }
let idade_ana = idades["Ana"]  // 25
idades["Carlos"] = 35          // insere ou atualiza

let lookup: dict<int, str> = { 1: "um", 2: "dois" }

Tipos Nominais (Classes)

Snask usa tipagem nominal baseada em classes. Dois tipos são compatíveis apenas se um é subtipo do outro por herança explícita com extends. 

class Animal {
    let idade: int = 0
}

class Dog extends Animal {
    let nome: str = "rex"
}

let animal: Animal = new Dog()  // Dog é subtipo de Animal
let dog: Dog = new Dog()

// animal.nome     // erro de tipo: Animal não tem campo 'nome'
// let err: Dog = animal  // erro: Animal não é subtipo de Dog

Propriedades e métodos

Campos de classe são acessados com . e são imutáveis por padrão (a menos que declarados com mut). Métodos são funções declaradas dentro do corpo da classe e têm acesso aos campos da instância. 

class Contador {
    let valor: int = 0

    fun incrementar() {
        valor = valor + 1
    }

    fun total() -> int {
        return valor
    }
}

let c = new Contador()
c.incrementar()
print("{c.total()}\n")

Anotações de Tipo

Snask permite anotar explicitamente o tipo de qualquer declaração usando a sintaxe : Tipo. A anotação é opcional na maioria dos contextos e pode ser usada para documentação ou para restringir a inferência. 

let nome: str = "ola"
let idade: int = 25
let altura: float = 1.75
let ativo: bool = true

fun soma(a: int, b: int) -> int {
    return a + b
}

let lista: list<str> = ["a", "b"]
let mapa: dict<str, int> = { "chave": 1 }

Anotações de tipo são obrigatórias em parâmetros de função e no tipo de retorno quando a função é exportada de um módulo.

Inferência de Tipos

Quando o tipo não é explicitamente anotado, Snask infere o tipo a partir do valor da expressão. A inferência é local e monomórfica. 

let x = 10        // infere int
let y = 3.14      // infere float
let s = "Snask"   // infere str
let b = true      // infere bool
let nums = [1, 2, 3]   // infere list<int>

let resultado = soma(10, 20)  // infere int pelo retorno de soma()

A inferência funciona em declarações locais, retorno de funções e literais de coleções homogêneas. Coleções vazias ([] ou {}) exigem anotação explícita.

Tipos na Interpolação

A interpolação de strings "{expr}" aceita qualquer expressão cujo tipo implemente a conversão para string. Todos os tipos primitivos, coleções e classes têm conversão automática. 

let nome = "Snask"
let versao = 5
let ativo = true
let preco = 29.99

let msg = "{nome}: v{versao}, ativo={ativo}, preco={preco}"
// "Snask: v5, ativo=true, preco=29.99"

Tipos de Recurso OM

O modelo OM (Owned Memory) introduz tipos de recurso que funcionam como handles opacos. Eles não podem ser copiados implicitamente — a transferência de posse é explícita. 

zone "recurso" {
    let handle = abre_arquivo("dados.bin")
    usa_recurso(handle)
}  // handle é liberado automaticamente ao sair da zona

Tipos de recurso OM são criados por funções que retornam handles gerenciados. O ciclo de vida é delimitado pela zona de alocação. A implementação formal de escape analysis e borrow checking ainda está em desenvolvimento.

experimental Tipos de recurso OM — arquitetura existe, faltam regras estáticas completas.
