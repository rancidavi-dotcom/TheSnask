Estrutura de Arquivo

Um executável Snask usa a classe main com o método start() como ponto de entrada. Não é necessário declarar fun main() — o runtime procura por class main { fun start() } automaticamente. 

class main {
    fun start() {
        print("Olá, Snask!\n")
    }
}

O método start() não recebe parâmetros e não tem valor de retorno. Todo o código de inicialização do programa deve estar dentro dele ou ser chamado a partir dele.

let x = 10  // comentário no final da linha

Variáveis

Snask oferece três formas de declaração, cada uma com semântica distinta:

Palavra-chave Mutabilidade Escopo Exemplo
let Imutável — não pode ser reatribuída Local / bloco let nome = "Snask"
mut Mutável — pode ser reatribuída Local / bloco mut contador = 0
const Imutável — valor constante em tempo de compilação Módulo const MAX = 100 
let nome: str = "Snask"
let idade = 5            // inferido como int

mut contador = 0
contador = contador + 1   // OK: mut permite reatribuição

const TAMANHO_MAX = 1024  // constante de módulo

Blocos

Snask aceita ambas as formas de delimitação de blocos: indentação significativa (como Python) e chaves { } (como C). Você pode misturar as duas abordagens no mesmo arquivo.

Blocos por indentação

if ready
    print("pronto\n")
    processa()

Blocos por chaves

if ready {
    print("pronto\n")
    processa()
}

A indentação padrão é de 4 espaços. Tabulações também são aceitas, mas não devem ser misturadas com espaços no mesmo bloco.

Números, Strings e Interpolação

Números

Snask tem dois tipos numéricos primários: int (inteiro com sinal, tamanho da palavra da máquina) e float (ponto flutuante de 64 bits). Literais numéricos podem usar _ como separador visual. 

let a = 42
let b = 1_000_000
let c = 0xFF        // hexadecimal
let d = 0b1010      // binário
let e = 3.14        // float
let f = 1.5e-10     // notação científica

Strings

Strings são delimitadas por aspas duplas ". Caracteres de escape usam barra invertida: \n, \t, \\, \".

Interpolação

Snask suporta interpolação de strings com a sintaxe {expressão} dentro de strings delimitadas por aspas duplas. O runtime converte automaticamente o valor para string. 

let nome = "Snask"
let versao = 5
let msg = "{nome}: versão {versao}\n"
// resultado: "Snask: versão 5\n"

Controle de Fluxo

if / else

if temperatura > 30 {
    print("quente\n")
} else if temperatura > 20 {
    print("agradável\n")
} else {
    print("frio\n")
}

while

mut i = 0
while i < 10 {
    print("{i}\n")
    i = i + 1
}

for in

O loop for in itera sobre elementos de uma coleção ou range.

let nomes = ["Ana", "Bia", "Carlos"]
for nome in nomes {
    print("{nome}\n")
}

for i in range(0, 5) {
    print("{i}\n")
}

Funções

Funções são declaradas com a palavra-chave fun, seguidas do nome, parâmetros entre parênteses e tipo de retorno opcional. O corpo pode ser delimitado por indentação ou chaves. 

fun saudacao(nome: str) -> str {
    return "Olá, {nome}!"
}

fun dobro(x: int) -> int {
    return x * 2
}

fun log(msg: str) {
    print("[log] {msg}\n")
}

let msg = saudacao("Mundo")  // "Olá, Mundo!"
let d = dobro(21)            // 42

Funções sem tipo de retorno explícito não retornam valor (equivalente a void). O tipo de retorno é inferido quando possível, mas a anotação explícita com -> Tipo é recomendada em APIs públicas.

Classes

Snask oferece um sistema de classes nominal com herança simples. Classes são declaradas com class, campos com let e métodos com fun. 

class Animal {
    let idade: int = 0

    fun envelhecer() {
        idade = idade + 1
    }
}

class Dog extends Animal {
    let nome: str = "rex"

    fun latir() {
        print("{nome} está latindo!\n")
    }
}

let animal: Animal = new Dog()
animal.envelhecer()

A herança usa extends e segue o princípio de subtipagem nominal: uma instância de Dog pode ser atribuída a uma variável do tipo Animal. A criação de instâncias usa a palavra-chave new.

estavel Classes, herança com extends e new.

Coleções

Snask tem duas coleções genéricas nativas: list<T> e dict<K, V>. Ambas suportam literais, indexação e atribuição.

list<T>

let vazia: list<int> = []
let numeros: list<int> = [1, 2, 3, 4, 5]
let primeiro = numeros[0]    // 1
numeros[2] = 42              // substitui o terceiro elemento

dict<K, V>

let vazio: dict<str, int> = {}
let idades: dict<str, int> = { "Ana": 25, "Bia": 30 }
let idade_ana = idades["Ana"]  // 25
idades["Carlos"] = 35          // adiciona ou substitui

Módulos

Snask suporta importação de módulos de duas formas: 

import "sfs"                   // importa todo o módulo sfs
from utils import math_helpers  // importa apenas math_helpers do módulo utils

O import simples é estável para arquivos locais e módulos nativos. O from ... import ... é funcional mas ainda está em fase de amadurecimento para casos complexos como importação seletiva com aliasing.

Safe / Unsafe

Snask usa a anotação @unsafe para demarcar regiões que realizam operações de memória crua. Fora dessas regiões, operações como mem_alloc, mem_free e ponteiros são bloqueadas pelo compilador. 

@unsafe {
    let ptr = mem_alloc(64)
    mem_write_u8(ptr, 0, 0xFF)
    let val = mem_read_u8(ptr, 0)
    mem_free(ptr)
}

O gate @unsafe funciona como uma barreira explícita: qualquer código que precise acessar memória bruta, chamar FFI inseguro ou manipular ponteiros deve estar dentro de um bloco @unsafe.

estavel @unsafe gate para operações de memória crua.

Integração OM

O modelo OM (Owned Memory) da Snask permite gerenciamento de recursos com zonas de alocação explícitas. A sintaxe zone "nome" { } define uma zona de memória com ciclo de vida delimitado. 

zone "render" {
    let buffer = aloca_recursos()
    renderiza(buffer)
}  // buffer é liberado ao sair da zona

A integração com C usa import_c_om para importar cabeçalhos C com adaptadores OM. 

import_c_om "SDL2/SDL.h"

parcial OM architecture existe, mas escape analysis e borrow checking ainda estão em desenvolvimento.

Perfis de Compilação

Snask oferece dois perfis de compilação que controlam quais funcionalidades da linguagem estão disponíveis:

Perfil Flag Descrição
humane (padrão) Perfil de alto nível com gerenciamento automático de memória. Acesso a IO, strings, coleções e GUI. Operações de memória crua são bloqueadas fora de @unsafe.
systems --profile systems Perfil de baixo nível para sistemas embarcados, NES e programação próxima ao hardware. Acesso completo a tipos de tamanho fixo (u8, i8, etc.), operações de bits, memória crua e interrupções. 
# Compilar com perfil systems
snask build --profile systems meu_programa.snask

estavel Perfil humane (padrão).

parcial Perfil systems — primitivas existem, layout/ABI ainda em fechamento.
