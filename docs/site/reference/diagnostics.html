Humane Diagnostics

Diagnósticos — Erros e Explain

A Snask quer que o erro seja parte da experiência de aprendizado, não uma parede de texto. Cada diagnóstico é projetado para ser legível, localizado, e acionável — com código, localização exata, nota contextual e sugestão de correção.

Filosofia de Erros

Na Snask, mensagens de erro são tratadas como parte integrante da experiência de desenvolvimento. Cada diagnóstico segue três princípios:

  • Localizado: o erro aponta para o arquivo, linha e coluna exatos, com uma seta (^) marcando o local preciso no código fonte.
  • Contextual: uma nota explica por que o erro ocorreu, frequentemente referenciando a abertura de um bloco, o tipo esperado ou a declaração anterior relevante.
  • Acionável: uma sugestão (help) oferece o caminho mais provável para corrigir o problema.

O código de erro (Snnnn) é estável dentro do mesmo release menor — você pode procura-lo na documentação ou usar snask explain para obter uma explicação detalhada.

Formato de um Diagnóstico

O formato completo de um diagnóstico Snask segue esta estrutura: 

error[S1002]: missing closing `)`
 --> hello.snask:3:22
  |
3 |         print("Hello"
  |                      ^ expected `)` here
note: unclosed '(' opened at 3:14
help: You probably missed a closing ')'.

Cada parte do diagnóstico tem um propósito específico:

Parte Descrição
error[S1002] Código do erro. O prefixo S identifica Snask; o número de 4 dígitos categoriza o tipo de erro.
missing closing `)` Mensagem principal, em inglês (localização pt-BR em planejamento).
--> hello.snask:3:22 Localização: arquivo, linha e coluna.
| + código + ^ Trecho do código fonte com a seta apontando o local exato do erro.
note: Informação contextual adicional sobre a causa raiz.
help: Sugestão de correção.

Como Ler um Diagnóstico

Considere o arquivo calc.snask com o seguinte conteúdo: 

class main {
    fun start() {
        let x = 10
        let y = 20
        let z = x + 
    }
}

O compilador produzirá um diagnóstico apontando a expressão incompleta: 

error[S1005]: unexpected end of expression
 --> calc.snask:5:21
  |
5 |         let z = x +
  |                     ^ expected expression after operator
note: binary operator '+' requires a right-hand operand
help: Add a value after the '+', for example: let z = x + y
  1. Identifique o código do erro (S1005) e a mensagem principal.
  2. Olhe a localização: linha 5, coluna 21 do arquivo calc.snask.
  3. Examine o código fonte exibido com a seta ^.
  4. Leia a nota para entender o contexto do erro.
  5. Siga a sugestão help para corrigir.

Faixas de Códigos de Erro

Os códigos de erro Snask são organizados em faixas numéricas que indicam a fase do compilador onde o erro foi detectado:

Faixa Área Status Quantidade aproximada
S0000 Erro desconhecido / fallback estavel 1
S1000–S1999 Parser e sintaxe estavel ~21 códigos
S2000–S2999 Semântica e tipos parcial ~19 códigos
S8000–S8999 Perfis de build e restrições de runtime parcial Em desenvolvimento
S9000–S9999 Toolchain, build e políticas parcial Em desenvolvimento

Cada faixa reserva espaço para expansão futura. Códigos dentro de S1000–S1999 são considerados estáveis; as demais faixas ainda podem receber novos códigos e ajustes nas mensagens.

Diagnósticos do Parser

Os diagnósticos de parser cobrem erros sintáticos: tokens inesperados, delimitadores não fechados, indentação inválida e estrutura de bloco incorreta. Esta é a camada mais madura do sistema de diagnósticos, com aproximadamente 21 códigos de erro ativos.

Exemplos comuns

Código Mensagem Causa comum
S1001 unexpected token Um token não esperado foi encontrado (ex.: } sem {)
S1002 missing closing ) Parêntese aberto mas não fechado
S1003 missing closing } Chave aberta mas não fechada
S1004 unclosed string literal String sem aspas de fechamento
S1005 unexpected end of expression Expressão incompleta (ex.: operador sem operando)
S1006 invalid indentation Mistura de espaços e tabs ou indentação inconsistente
S1007 expected block after header Header de controle (if/while) sem corpo 
// Exemplo que gera S1002:
class main {
    fun start() {
        print("Hello"
    }
}
// error[S1002]: missing closing `)`

Diagnósticos Semânticos

Erros semânticos e de tipo são detectados após o parser, durante a análise de tipos e resolução de nomes. Esta camada está em amadurecimento — os diagnósticos existem mas ainda estão recebendo sugestões contextuais mais refinadas.

Exemplos comuns

Código Mensagem Causa comum
S2001 undefined variable Uso de variável que não foi declarada no escopo atual
S2002 type mismatch Tipo esperado não corresponde ao tipo fornecido
S2003 cannot assign to immutable variable Tentativa de reatribuir uma variável let
S2004 undefined function Chamada de função que não existe no escopo
S2005 wrong number of arguments Número de argumentos não corresponde à declaração da função
S2006 undefined field Acesso a campo que não existe na classe
S2007 cannot use unsafe operation outside @unsafe block Operação de memória crua fora de região @unsafe
S2008 return type mismatch Valor retornado não corresponde ao tipo de retorno declarado 
// Exemplo que gera S2002:
let x: int = "texto"
// error[S2002]: type mismatch: expected `int`, found `str`

parcial Diagnósticos semânticos existem (~19 códigos), mas sugestões contextuais ainda estão amadurecendo.

Perfis e Restrições de Build

A faixa S8000–S8999 cobre erros relacionados a perfis de compilação (humane vs systems) e restrições de runtime. Por exemplo, usar uma função do perfil systems sem a flag --profile systems produz um erro nesta faixa. 

// Tentativa de usar bit_test sem perfil systems
let result = bit_test(0xFF, 3)
// error[S8001]: `bit_test` requires profile `systems`

parcial Diagnósticos de perfil de build — cobertura ainda está sendo expandida.

Toolchain, Build e Políticas

A faixa S9000–S9999 é reservada para erros da toolchain: arquivo não encontrado, falha de linkagem, configuração inválida de projeto, e violações de política (ex.: uso de feature experimental sem opt-in explícito). 

snask build programa.snask
// error[S9001]: file not found: `programa.snask`

parcial Diagnósticos de toolchain — área mais recente, ainda em desenvolvimento ativo.

Comando explain

O comando snask explain fornece uma explicação detalhada para qualquer código de erro, incluindo descrição, causas comuns e exemplos de código incorreto vs. corrigido. 

snask explain S1002
snask explain S2002
snask explain S8001

A saída do explain inclui:

  • O código e nome do erro
  • Uma descrição detalhada do problema
  • O que costuma causar esse erro
  • Exemplo de código que produz o erro
  • Exemplo do código corrigido
  • Referência cruzada para documentação relacionada
$ snask explain S1002

S1002 – missing closing `)`

  This error occurs when the parser finds an opening parenthesis
  without a matching closing parenthesis before the end of the
  expression or statement.

Common causes:
  • Forgot to close a function call: `print("hello`
  • Nested parentheses not properly closed: `(a + (b * c)`
  • Multi-line expression where the closing paren was omitted

Example:
  ❌ print("Hello"
  ✅ print("Hello")

Status Atual

O sistema de diagnósticos da Snask está em desenvolvimento ativo. Abaixo, o resumo do status de cada camada:

Camada Status Observação
Parser diagnostics (S1000–S1999) estavel Base estável com ~21 códigos; mensagens claras e localização precisa
Semantic diagnostics (S2000–S2999) parcial ~19 códigos existem; sugestões contextuais e cobertura de casos complexos ainda amadurecendo
OM / linker diagnostics experimental Ainda maturing; erros de borrow/ownership podem ser menos específicos
Profiles & runtime (S8000–S8999) parcial Cobertura básica; expansão em progresso
Toolchain & policies (S9000–S9999) parcial Área mais recente; novos códigos sendo adicionados
Nota: Se você encontrar um diagnóstico que poderia ser mais claro, abra uma issue no repositório da Snask. Mensagens de erro são consideradas parte da experiência do usuário e melhorias são bem-vindas.
